正则表达式

原子表

原子表中有些正则字符不需要转义 [.+()]

原子表中. 就是小数点，+就是加号

原子组 ()

  • 如果一次要匹配多个元子，可以通过元子组完成
  • 原子组与原子表的差别在于原子组一次匹配多个元子，而原子表则是匹配任意一个字符
  • 元字符组用 () 包裹

下面使用原子组匹配 h 标签

const hd = `<h1>houdunren.com</h1>`;
console.log(/<(h[1-6])>([\s\S]*)<\/\1>/.test(hd)); //true

没有添加 g 模式修正符时只匹配到第一个，匹配到的信息包含以下数据

变量 说明
0 匹配到的完整内容
1,2…. 匹配到的原子组
index 原字符串中的位置
input 原字符串
groups 分组别名

引用分组

\xx 在匹配时引用原子组， $xx 指在替换时使用匹配的组数据。下面将h``标签替换为p`标签

let hd = `
  <h1>houdunren</h1>
  <span>后盾人</span>
  <h2>hdcms</h2>
`;

let reg = /<(h[1-6])>([\s\S]*)<\/\1>/gi;
console.log(hd.replace(reg, `<p>$2</p>`));
  • $1取的是(h[1-6])的内容（第一个括号里的内容）
  • $2取的是([\s\S]*)的内容
  • 顺序就是按照(的出现顺序

(?:xxx)不记录组编号

//正则表达式
let reg = /https?:\/\/((?:\w+\.)?\w+\.(?:com|org|cn))/gi

https?表示s可以有可以没有，?表示0个或1个
(?:\w+\.)表示这个原子组不会被记录到编号里，就是不会被记录到$2,也无法使用/2来引用

(?<name>) 给原子组命名

(?<name>.*?) 给组命名为name
使用时用$<name>可以获取到匹配内容，代替以前用数组索引的$1

下面将h标签替换为p标签

let hd = `
  <h1>houdunren</h1>
  <span>后盾人</span>
  <h2>hdcms</h2>
`;
let reg = /<(?<tag>h[1-6])>(?<con>[\s\S]*)<\/\1>/gi;
console.log(hd.replace(reg, `<p>$<con></p>`));

重复匹配

  • +1个或多个
  • *0个或多个
  • ?0个或1个
  • {n}重复n次
  • {n,}重复n次或更多次
  • {n,m}重复n到m次

?禁止贪婪

let reg = /\w+?/gi 匹配1个
let reg = /\w??/gi 匹配0个
let reg = /\w{2,8}?/gi 匹配2个

将所有span更换为h4 并描红，并在内容前加上 后盾人-

<body>
  <main>
    <span>houdunwang</span>
    <span>hdcms.com</span>
    <span>houdunren.com</span>
  </main>
</body>
<script>
  const main = document.querySelector("main");
  const reg = /<span>([\s\S]+?)<\/span>/gi;
  main.innerHTML = main.innerHTML.replace(reg, (v, p1) => {
    //reg 表示正则匹配的所有内容
    //v 表示匹配到的内容
    //p1 表示第一个原子组([\s\S]+?)匹配的内容
    console.log(p1);
    return `<h4 style="color:red">后盾人-${p1}</h4>`;
  });
</script>

全局匹配

str.match

使用match 全局获取页面中标签及其内容，但并不会返回匹配细节

<body>
  <h1>houdunren.com</h1>
  <h2>hdcms.com</h2>
  <h1>后盾人</h1>
</body>

<script>
  function elem(tag) {
    const reg = new RegExp("<(" + tag + ")>.+?<\.\\1>", "g");
    return document.body.innerHTML.match(reg);
  }
  console.table(elem("h1"));
</script>

str.matchAll

在新浏览器中支持使用 matchAll 操作，并返回迭代对象

let str = "houdunren";
let reg = /[a-z]/ig;
for (const iterator of str.matchAll(reg)) {
  console.log(iterator);
}

reg.exec

使用 g 模式修正符并结合 exec 循环操作可以获取结果和匹配细节

<body>
  <h1>houdunren.com</h1>
  <h2>hdcms.com</h2>
  <h1>后盾人</h1>
</body>
<script>
  function search(string, reg) {
    const matchs = [];
    while ((data = reg.exec( string))) {
      matchs.push(data);
    }
    return matchs;
  }
  console.log(search(document.body.innerHTML, /<(h[1-6])>[\s\S]+?<\/\1>/gi));
</script>

String 提供的支持正则表达式的方法

search

search() 方法用于检索字符串中指定的子字符串，也可以使用正则表达式搜索
返回值为索引位置

let str = "houdunren.com";
console.log(str.search("com"));
console.log(str.search(/\.com/i));

match matchAll

split

用于使用字符串或正则表达式分隔字符串，下面是使用字符串分隔日期

let str = "2023-02-12";
console.log(str.split("-")); //["2023", "02", "12"]

//如果日期的连接符不确定，那就要使用正则操作了
console.log(str.split(/-|\//));

replace

$

“$&“匹配到的内容
“ $` “ 匹配到的内容的左边内容
“$'“匹配到的内容的右边内容
“$n“第 n 个括号（原子组）匹配的字符串。索引是从1开始

回调函数

为所有标题添加上 hot 类

<body>
  <div class="content">
    <h1>后盾人</h1>
    <h2>houdunren.com</h2>
    <h1>后盾人</h1>
  </div>
</body>
<script>
  let content = document.querySelector(".content");
  let reg = /<(h[1-6])>([\s\S]*?)<\/\1>/gi;
  content.innerHTML = content.innerHTML.replace(
    reg,
    (
      search, //匹配到的字符
      p1, //第一个原子组
      p2, //第二个原子组
      index, //索引位置
      source //原字符
    ) => {
      return `
    <${p1} class="hot">${p2}</${p1}>
    `;
    }
  );
</script>

正则的方法

test

检测输入的邮箱是否合法，返回true或false

<body>
  <input type="text" name="email" />
</body>

<script>
  let email = document.querySelector(`[name="email"]`);
  email.addEventListener("keyup", e => {
    console.log(/^\w+@\w+\.\w+$/.test(e.target.value));
  });

exec

  • 会返回细节
  • 不使用g 修饰符时与 match 方法使用相似，使用 g 修饰符后可以循环调用直到全部匹配完。
  • 使用 g 修饰符多次操作时使用同一个正则，即把正则定义为变量使用
  • 使用 g 修饰符最后匹配不到时返回 null

断言

(?=xxx) 后面是

/hello(?=world)/ 匹配后面紧跟world的hello，类似条件语句
匹配到的内容不包括world

(?<=xxx) 前面是

/(?<=hello)world/ 匹配前面是hello的world，匹配到的内容不包括hello

(?!xxx) 后面不是

/[a-z]+(?!\d+)/ 匹配后面不是数字的字母

(?<!xxx) 前面不是

/(?<!\d+)[a-z]+/ 匹配前面不是数字的字母

元子字符

/^ $/开始-结束

/s/空白 - /S/除了空白

空格，换行/n 都是空白

.除了换行符外任何字符

/.+/s可以匹配到换行以后的东西，因为s表示匹配单行模式

/w字母 数字 下划线 - /W除了字母 数字 下划线

/d/ 数字 /D/ 除了数字

空格是普通字符 /\d+/s-\s\d+/等于/\d+ - \d+/

匹配所有字符 /[\s\S]/ /[\d\D]/ 包括换行

模式修饰

i不缺分大写

g匹配全部

m每行单独处理

u匹配宽字节

y在匹配不到时停止匹配

lastIndex 返回或者设置正则表达式开始匹配的位置

RegExp对象lastIndex 属性可以返回或者设置正则表达式开始匹配的位置
必须结合 g 修饰符使用
对 exec 方法有效
匹配完成时，lastIndex 会被重置为0
